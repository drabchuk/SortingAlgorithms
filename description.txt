Selection sort

	complexity:	N^2

	compares:	N^2/2
	exchanges:	N

	+:	
		Простой в реализации
		N перестановок
	
	-:
		Квадратичная сложность
		Плохой для упорядоченных массивов

	usage:
		Быстрая и безошибочная реализация

Insertion sort

	complexity:	N^2

	compares:	N^2/2
	exchanges:	N^2/2

	+:	
		Простой в реализации
		Подходит для частично упорядоченных
	
	-:
		Квадратичная сложность
		Плохой для обратно упорядоченных массивов

	usage:
		Частично упорядоченные массивы

Shell sort

	complexity:	N^(3/2)

	compares:	N^(3/2)
	exchanges:	N

	+:	
		Простой в реализации
		Подходит для частично упорядоченных
		Быстрое выполнение для коротких массивов
	
	-:
		Сложность выше чем у других, более сложных алгоритмов


	usage:
		Быстро написать хороший алгоритм
		Для коротких массивов, до ~100.000 быстрее сортировки слиянием


Merge sort

	complexity:	N*log(N)

	+:
		Ассимптотически оптимальный алгоритм,
		гарантия выполнения за некоторое время

	-:
		Требует выделения дополнительной памяти


	usage:
		Быстрая сортировка больших массивов, при наличии доп. памяти

Quick sort

	complexity:	N*log(N)

	+:
		Часто, самый быстрый алгоритм.
		Не требует дополнительной памяти

	-:
		Чувствительный алгоритм, в некоторых случаях выполняется долго.


	usage:
		Быстрая сортировка больших массивов.
